# UX Designer Brief - API Integration Training

## Project Overview
Educational web app teaching developers API integration patterns through interactive demos. Built with Next.js, deployed at [your-domain.com].

**Stack:** Next.js 14, React, TypeScript, Tailwind CSS  
**Target:** Junior to mid-level developers learning API patterns

---

## Your Mission
Improve the user experience to help developers learn API integration faster and with more confidence.

**You decide:**
- What to audit and how
- Which heuristics to apply
- What to prioritize
- How to measure success

We trust your expertise.

---

## Target Users

### Primary Personas
1. **Junior Developer (1-2 years)** - Learning API patterns for first time
2. **Mid-level Developer (3-5 years)** - Needs quick reference and best practices

### User Goals
- Learn API patterns quickly
- Practice with interactive demos
- Reference documentation when implementing
- Understand when to use each pattern

---

## Current State

### What's Built
- 4 phases: Integration Mindset → Third-Party Integrations → Internal APIs → Production
- Interactive demos: REST, OAuth2, Circuit Breaker, Retry logic, etc.
- Documentation modals with "Pareto Principle" summaries
- Mobile app (React Native/Expo) in parallel

### Known Issues
Users have mentioned:
- Navigation could be clearer
- Sometimes feeling lost in the content hierarchy
- Mobile experience needs work
- No search functionality

**But you'll discover more. We're counting on it.**

---

## Success Criteria

### What Good Looks Like
- Users complete demos without external help
- Users confidently implement patterns after training
- High task success rate, low error rate
- Users recommend to colleagues

### Constraints
- Must work on mobile
- Must be accessible (WCAG 2.1 AA)
- Must load fast (< 2 sec per page)
- Dark mode is primary (light mode optional)
