# Self-hosted AI runner requirements

These requirements apply to the isolated Linux host intended to run trusted Architecture Intelligence jobs with Ollama. This repository is public; do not route fork pull requests or other untrusted code to this host.

## Host

| Requirement | Minimum | Recommended |
|-------------|---------|-------------|
| Operating system | Ubuntu 22.04 LTS x64 | Ubuntu 24.04 LTS x64 |
| CPU | 4 modern cores | 8+ cores |
| Memory | 16 GB RAM | 32 GB RAM |
| Free disk | 25 GB | 50 GB SSD |
| GPU | Not required | NVIDIA GPU with 8+ GB VRAM |
| Availability | Online during requested jobs | Dedicated always-on or scheduled host |

`qwen2.5-coder:7b` occupies approximately 4.7 GB. The remaining capacity is for the runner application, Node.js tool cache, repository workspaces, npm dependencies, Next.js builds, logs, and temporary files.

## Network

- Outbound HTTPS on TCP 443 to GitHub and GitHub-owned download endpoints.
- Outbound HTTPS to the npm registry and Ollama model registry.
- Inbound SSH restricted to an administrator-controlled IP or private management network.
- Ollama bound only to `127.0.0.1:11434`; never expose port 11434 publicly.
- No route to production databases, internal administration networks, or personal devices.

## Accounts and permissions

- A non-root `actions` account owns `/opt/actions-runner` and executes workflow jobs.
- The standard `ollama` service account owns the model directory.
- The `actions` account does not receive passwordless `sudo`.
- Do not add `actions` to the `docker` group or expose a Docker socket for this workflow.
- Keep GitHub runner registration files readable only by the `actions` account.

## Software dependencies

| Dependency | Installation source | Purpose |
|------------|---------------------|---------|
| GitHub Actions runner | Commands generated by GitHub repository settings | Receives and executes jobs |
| Git, curl and CA certificates | Ubuntu packages | Checkout and secure downloads |
| Build tools, tar, gzip and xz | Ubuntu packages | Native npm modules and action archives |
| Node.js 22 | `actions/setup-node` in the workflow | Project scripts, tests and build |
| Ollama | Official Ollama Linux installer | Local inference server |
| `qwen2.5-coder:7b` | Ollama model registry | Architecture review model |
| Ollama provider adapter | Future repository integration | OpenAI-compatible client for local Ollama |

Docker and a live PostgreSQL database are not requirements for the Architecture Intelligence job. Keep the full Docker/PostgreSQL smoke suite on GitHub-hosted runners.

## Security policy

- Start with `workflow_dispatch` and protected-branch `push` events only.
- Do not assign this runner to arbitrary `pull_request` jobs.
- Use runner labels `ollama` and `ai-review` so unrelated jobs cannot select it accidentally.
- Do not store OpenAI, Neon, Vercel, production database, SSH, or personal credentials on the host.
- Remove the job workspace after every run or replace the machine with an ephemeral image.
- Apply Ubuntu and Ollama security updates regularly and monitor runner version support.

GitHub warns that self-hosted runners can be persistently compromised by untrusted workflow code. Isolation is a requirement, not an optional hardening step.
