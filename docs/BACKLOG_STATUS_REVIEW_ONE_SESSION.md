# Backlog status review — one session

Use this for **one** standing review (~15–30 min). Open the [work tracker](https://www.notion.so/6ea1b2867262468fbb15da5984602d5b) and work top to bottom.

## 1. Backlog

- [ ] Removed or merged duplicate / abandoned rows  
- [ ] Valid items still vague → left in **Backlog** with a short note, or groomed → **Ready**  
- [ ] Nothing important hiding without **Work Type** / minimal description  

**Notes**

---

## 2. Ready

- [ ] Top of queue ordered (what we pull next is obvious)  
- [ ] Each **Ready** row has clear scope in one place (description or page)  
- [ ] **Ready** count stays small; excess pushed back to **Backlog**  

**Notes**

---

## 3. In Progress (`Board Status`)

- [ ] Every row has an owner (explicit or obvious from comments)  
- [ ] Stale rows decided: continue / hand off → **Ready** / park → **Backlog** + reason  
- [ ] Mirrored **Status** field is **In progress** (see [FEATURE_MANAGEMENT](FEATURE_MANAGEMENT.md))  

**Notes**

---

## 4. In Review (`Board Status`)

- [ ] Every row has a **PR URL in Notes** (or moved back to **In Progress** / **Ready** until it does)  
- [ ] Stuck items have a blocker comment or split follow-up row  

**Notes**

---

## 5. Done (`Board Status`)

- [ ] Last period’s **Done** rows have PR URL in **Notes** or “N/A” reason  
- [ ] Optional: deploy / release tag noted where you care about audit  

**Notes**
