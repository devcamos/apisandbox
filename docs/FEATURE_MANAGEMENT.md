# Feature Management

Feature tracking for `apisandbox` now lives in Notion.

## Source of truth

- Repo hub: https://www.notion.so/3417233a96ec81ddbae4dcd29a68b775
- Work tracker: https://www.notion.so/6ea1b2867262468fbb15da5984602d5b

## How to work

1. Search the tracker before starting a feature, fix, or maintenance task.
2. If no ticket exists, create a new item in the tracker.
3. Use the Notion page inside the tracker row as the ticket body.
4. Keep status, blockers, verification notes, and repo links updated in that page.
5. Link implementation summaries, fix guides, screenshots, and reports from the Notion ticket instead of creating separate repo-side feature specs.

## Notes

- This repo does not currently use a top-level `docs/features/` workflow.
- Jira URLs can be added to the `Jira Ticket` field in Notion when available.
