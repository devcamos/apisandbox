# Vercel Environment Variables Reference

Complete list of all environment variables needed for Vercel deployment.

## 📋 Quick Reference

**Location**: Vercel Dashboard → Your Project → Settings → Environment Variables

**How to Set**: 
1. Go to your project in Vercel Dashboard
2. Click **Settings** → **Environment Variables**
3. Add each variable below
4. Apply to: **Production**, **Preview**, **Development** (or specific environments)

---

## 🔴 Required Variables

### Database

| Variable | Value | Description | Source |
|----------|-------|-------------|--------|
| `DATABASE_URL` | `$POSTGRES_PRISMA_URL` | PostgreSQL connection string for Prisma | **Auto-generated** by Vercel Postgres (see below) |

**How to get**: 
1. Create Vercel Postgres database (Storage tab → Create Database → Postgres)
2. Vercel automatically creates `POSTGRES_PRISMA_URL`
3. Set `DATABASE_URL=$POSTGRES_PRISMA_URL` (references the auto-generated variable)

**Alternative**: If using external PostgreSQL:
```
DATABASE_URL="postgresql://user:password@host:5432/database?sslmode=require"
```

### Authentication

| Variable | Value | Description | How to Generate |
|----------|-------|-------------|-----------------|
| `AUTH_SECRET` | `[32+ character string]` | NextAuth.js secret for JWT signing | `openssl rand -base64 32` |
| `AUTH_JWT_SECRET` | `[32+ character string]` | JWT for `/api/auth/register`, login, Google | `openssl rand -base64 32` |
| `NEXTAUTH_SECRET` | `[32+ character string]` | Alias for AUTH_SECRET (legacy support) | Same as `AUTH_SECRET` |

**Preview deployments:** set `AUTH_JWT_SECRET` (or `AUTH_SECRET`) for **Preview** on all branches, not only Production. Without it, signup/Google return `JWT secret is not configured`.

**Generate command**:
```bash
openssl rand -base64 32
```

**Example format** (DO NOT use this - generate your own):
```
AUTH_SECRET="[64-character-base64-string]"
```

**Important**: Always generate a unique secret for each environment. Never reuse secrets.

### Environment

| Variable | Value | Description |
|----------|-------|-------------|
| `NODE_ENV` | `production` | Node.js environment (Vercel may set this automatically) |

## 🟡 Optional Variables

### Application URL

| Variable | Value | Description |
|----------|-------|-------------|
| `NEXT_PUBLIC_APP_URL` | `https://your-app.vercel.app` | Public app URL for OAuth callbacks |

**Note**: Vercel automatically sets this, but you can override if needed.  
**Auto-detected from**: `VERCEL_URL` environment variable (set by Vercel automatically)

### OAuth Providers (Google)

| Variable | Value | Description | Setup |
|----------|-------|-------------|-------|
| `GOOGLE_CLIENT_ID` | `[from Google Cloud Console]` | Google OAuth Client ID | [Google Cloud Console](https://console.cloud.google.com/) |
| `GOOGLE_CLIENT_SECRET` | `[from Google Cloud Console]` | Google OAuth Client Secret | [Google Cloud Console](https://console.cloud.google.com/) |

**Setup Steps**:
1. Go to [Google Cloud Console](https://console.cloud.google.com/)
2. Create OAuth 2.0 credentials
3. Add authorized redirect URI: `https://your-app.vercel.app/api/auth/callback/google`
4. Copy Client ID and Client Secret to Vercel

**Without these**: Google OAuth login will be disabled. Email/password auth still works.

## 🔵 Auto-Generated by Vercel (Read-Only)

These are automatically set by Vercel. You don't need to set them manually:

| Variable | Description | Usage |
|----------|-------------|-------|
| `VERCEL` | Set to `1` when running on Vercel | Auto-detection |
| `VERCEL_URL` | Deployment URL (e.g., `apisandbox-abc123.vercel.app`) | Auto URL generation |
| `POSTGRES_URL` | Direct PostgreSQL connection | Not recommended for Prisma |
| `POSTGRES_PRISMA_URL` | Prisma-optimized connection (with pooling) | **Use this for DATABASE_URL** |
| `POSTGRES_URL_NON_POOLING` | Direct connection without pooling | Use for migrations only |

## 📝 Complete Vercel Setup Example

### Step 1: Create Vercel Postgres Database

1. Vercel Dashboard → Your Project → **Storage** tab
2. Click **"Create Database"** → **Postgres**
3. Name: `apisandbox-db`
4. Region: Choose closest to your users
5. Click **"Create"**

Vercel automatically creates: `POSTGRES_PRISMA_URL`, `POSTGRES_URL`, `POSTGRES_URL_NON_POOLING`

### Step 2: Add Environment Variables

In **Settings → Environment Variables**, add:

```
DATABASE_URL = $POSTGRES_PRISMA_URL
AUTH_SECRET = [generated-32-char-secret]
NEXTAUTH_SECRET = [same-as-auth-secret]
NODE_ENV = production
```

**Optional (for Google OAuth)**:
```
GOOGLE_CLIENT_ID = [your-google-client-id]
GOOGLE_CLIENT_SECRET = [your-google-client-secret]
NEXT_PUBLIC_APP_URL = https://your-app.vercel.app
```

### Step 3: Apply to Environments

- ✅ **Production**: Required for production deployments
- ⚠️ **Preview**: Optional, for preview deployments (pull requests)
- ⚠️ **Development**: Optional, for local development (if using Vercel CLI)

After setting variables:

- [ ] `DATABASE_URL` is set (references `$POSTGRES_PRISMA_URL`)
- [ ] `AUTH_SECRET` is 32+ characters
- [ ] `AUTH_SECRET` generated with `openssl rand -base64 32`
- [ ] `NODE_ENV=production` (optional, Vercel may set automatically)
- [ ] OAuth variables set (if using Google login)
- [ ] Variables applied to **Production** environment

## 🔒 Security Notes

1. **Never commit secrets to Git**: All secrets are in `.gitignore`
2. **Use Vercel's variable reference**: `DATABASE_URL=$POSTGRES_PRISMA_URL` (not copy-paste the URL)
3. **Rotate secrets regularly**: Especially `AUTH_SECRET` if compromised
4. **Use different secrets per environment**: Production, staging, development
5. **HTTPS is automatic**: Vercel enables HTTPS for all deployments
6. **Database SSL**: Vercel Postgres uses SSL by default

## 🐛 Troubleshooting

### "AUTH_SECRET is required" Error

**Problem**: NextAuth can't find `AUTH_SECRET`

**Solution**:
1. Verify `AUTH_SECRET` is set in Vercel Dashboard
2. Check it's applied to the correct environment (Production)
3. Redeploy after adding the variable

### Database Connection Error

**Problem**: Can't connect to database

**Solution**:
1. Verify `DATABASE_URL=$POSTGRES_PRISMA_URL` (references Vercel's variable)
2. Check Vercel Postgres database is active
3. Run migrations: `npx prisma migrate deploy`

### OAuth Callback Error

**Problem**: Google OAuth redirect fails

**Solution**:
1. Verify callback URL in Google Cloud Console: `https://your-app.vercel.app/api/auth/callback/google`
2. Check `GOOGLE_CLIENT_ID` and `GOOGLE_CLIENT_SECRET` are set
3. Ensure `NEXT_PUBLIC_APP_URL` matches your Vercel deployment URL

For post-incident env/build/auth alignment (health URLs, Prisma build flags, preview `AUTH_JWT_SECRET`, Google origins, E2E against prod), see **[PRODUCTION_CONFIG_SYNC.md](./PRODUCTION_CONFIG_SYNC.md)**. Update its **changelog** table when you change deploy contracts in a PR.

### Prisma on Vercel (required knowledge)

- Set `DATABASE_URL` to **`$POSTGRES_PRISMA_URL`** (not a pasted URL).
- Do **not** enable `PRISMA_GENERATE_DATAPROXY` for this project. `package.json` runs `env -u PRISMA_GENERATE_DATAPROXY prisma generate` so the runtime uses a **binary** engine with `postgresql://` URLs.
- Verify after deploy: `GET /api/health/db` → success.

### Post-deploy smoke

```bash
curl -sS "https://apisandbox-coral.vercel.app/api/health/db"
curl -sS "https://apisandbox-coral.vercel.app/api/health/auth"
PLAYWRIGHT_PROD_URL=https://apisandbox-coral.vercel.app npm run test:prod
```
